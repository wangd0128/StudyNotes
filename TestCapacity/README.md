# CLR创建一个对象的时候内存是如何分配的

*主要是看操作系统的位数,这里以64位操作系统举例

* 8个字节  同步块索引，CLR 建立一个同步块数组用于存放同步信息（包含线程的信息）,  当一个线程试图使用某个对象进入同步代码块的时候，首先CLR会检查该对象的
	同步块索引，如果为负数-1的时候就说明没有现成占用该对象，CLR就会去创建一个同步块将线程信息写进去，并将该同步块的索引赋值给该对象的同步块索引。
	如果为不为负数，就根据索引去找到对应的同步块，并检查同步块里面有没有线程信息，如果没有就将线程信息写进去。如果有则等待

* 8个字节  对象类型指针，一个指针8个字节，指向 system.type
* 8个字节  空的，内存对齐，类里面没有成员或者一个int 一个char 都是8 除非超过8个字节


这里和C++的区别是因为 前面16个字节都是CLR用的
